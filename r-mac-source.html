This page describes the steps involved in compiling R-devel on a Mac (OSX v10.12.3).

Preliminaries

First we need to install some tools and libraries.

  • Get Xcode from the App Store. (The page should look something like this.) The latest version of Xcode should install the command line tools by default.
    • Open a terminal window—search spotlight for 'terminal' if you've never needed terminal/bash before—and type:
      • clang++ -v
    • You should see something similar to
      • Apple LLVM version 8.0.0 (clang-800.0.42.1)
Target: x86_64-apple-darwin16.4.0
Thread model: posix
  • Download and install XQuartz. (You might need to restart your computer to complete the installation process.)
  • Download and install the Java SDK. (No need for the 'demos and samples' pack.)

The following items are from the R-devel optional library list.

  • Unfortunately, Xcode does not include a Fortran compiler, which we will need. The simplest way to obtain an R-compatible version of gfortran is to open a terminal window and type:
    • cd ~/Desktop
curl -O http://r.research.att.com/libs/gfortran-4.8.2-darwin13.tar.bz2
sudo tar fvxz gfortran-4.8.2-darwin13.tar.bz2 -C /

    (The 'sudo' command requires you to enter your password—this is because we are placing files into areas that are password restricted (/usr/local).)

  • We also need pkg-config, xz, and pcre:
    • cd ~/Desktop
curl -O http://r.research.att.com/libs/pkgconfig-0.28-darwin.13-x86_64.tar.gz
sudo tar fvxz pkgconfig-0.28-darwin.13-x86_64.tar.gz -C /
curl -O http://r.research.att.com/libs/xz-5.2.1-darwin.13-x86_64.tar.gz
sudo tar fvxz xz-5.2.1-darwin.13-x86_64.tar.gz -C /
curl -O http://r.research.att.com/libs/pcre-8.36-darwin.13-x86_64.tar.gz
sudo tar fvxz pcre-8.36-darwin.13-x86_64.tar.gz -C /
  • Finally, for creating images, we need JPEG, PNG, and TIFF libraries:
    • cd ~/Desktop
curl -O http://r.research.att.com/libs/jpeg-9-darwin.13-x86_64.tar.gz
sudo tar fvxz jpeg-9-darwin.13-x86_64.tar.gz -C /
curl -O http://r.research.att.com/libs/libpng-1.6.17-darwin.13-x86_64.tar.gz
sudo tar fvxz libpng-1.6.17-darwin.13-x86_64.tar.gz -C /
curl -O http://r.research.att.com/libs/tiff-4.0.3-darwin.13-x86_64.tar.gz
sudo tar fvxz tiff-4.0.3-darwin.13-x86_64.tar.gz -C /

Download and Unpack

Create a folder to work in. For example,

cd ~/Desktop
mkdir R

Then download and unpack the latest development version of R:

cd ~/Desktop/R
curl -O ftp://ftp.stat.math.ethz.ch/Software/R/R-devel.tar.gz
tar -zxvf R-devel.tar.gz

Configure

Open the 'config.site' file in '~/Desktop/R/R-devel' and change the contents to the following:

#! /bin/sh

CC=clang
CXX=clang++
F77=gfortran-4.8
FC=$F77
OBJC=clang
CFLAGS="-Wall -mtune=core2 -g -O2"
CXXFLAGS="-Wall -mtune=core2 -g -O2"
OBJCFLAGS="-Wall -mtune=core2 -g -O2"
F77FLAGS="-Wall -g -O2"
FCFLAGS=$F77FLAGS

PKG_CONFIG_PATH=/opt/X11/lib/pkgconfig:/usr/local/lib/pkgconfig:/usr/lib/pkgconfig

JAVA_HOME="$(/usr/libexec/java_home)"
JAVA_CPPFLAGS="-I/${JAVA_HOME}/include -I/${JAVA_HOME}/include/darwin"
JAVA_LD_LIBRARY_PATH="${JAVA_HOME}/jre/lib/server"
JAVA_LIBS="-L/${JAVA_HOME}/jre/lib/server -ljvm"

Open a terminal window and switch to the R-devel folder:

cd ~/Desktop/R/R-devel

Set the pkg-config path as follows:

export PKG_CONFIG_PATH=/opt/X11/lib/pkgconfig:/usr/local/lib/pkgconfig:/usr/lib/pkgconfig

Now we have two choices. If you want to link against the OpenBLAS library (assuming you have it...) run

./configure --prefix=/Builds/R-devel --without-x --without-internal-tzcode \
	--with-blas="-lopenblas" --with-lapack \
		--enable-R-shlib --enable-memory-profiling

or, to link against the Accelerate framework (which is pre-installed on all Macs), run

./configure --prefix=/Builds/R-devel --without-x --without-internal-tzcode \
	--with-blas="-framework Accelerate" --with-lapack \
		--enable-R-shlib --enable-memory-profiling

Build and Install

To build R-devel, use

cd ~/Desktop/R/R-devel
make

The build checks are run using:

make check

Finally, we need to choose an install location for R-devel; I use /Builds/R-devel, but I recommend installing to somewhere that doesn't require sudo privileges.

sudo make prefix=/Builds/R-devel install

Calling R-devel

For easy access to R-devel from a terminal window, add the following lines of code to your bash profile:

alias Rdevel="/where/I/installed/to/bin/R"

For example,

alias Rdevel="/Builds/R-devel/bin/R"

To run R-devel with sudo privileges, you need to also add

alias sudo='sudo '

To run R-devel from RStudio, use

export RSTUDIO_WHICH_R=/where/I/installed/to/bin/R
open -n -a RStudio
